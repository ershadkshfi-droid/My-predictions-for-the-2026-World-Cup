# توقعاتي لكأس العالم 2026 (My Predictions for World Cup 2026)

## نظرة عامة على المشروع (Project Overview)
تطبيق تفاعلي يتيح للمستخدمين توقع نتائج مباريات كأس العالم 2026، وتكوين دوريات خاصة للتنافس مع الأصدقاء، وجمع النقاط بناءً على دقة التوقعات، وتصدر لوحة المتصدرين.

---

## 1. البنية الهندسية للبرمجيات (Software Architecture)
يعتمد النظام على معمارية الخدمات المصغرة / السحابية (Cloud-Native Architecture) ليكون مرناً وقابلاً للتوسع:
- **الواجهة الأمامية (Frontend):** تطبيق صفحة واحدة (SPA) يوفر تجربة مستخدم سريعة ومتجاوبة.
- **الواجهة الخلفية (Backend):** واجهة برمجة تطبيقات (RESTful API) تتعامل مع منطق الأعمال المعقد وحساب النقاط.
- **قاعدة البيانات (Database):** قاعدة بيانات سحابية لتخزين بيانات المستخدمين، التوقعات، وجدول المباريات.
- **خدمات التوثيق (Authentication):** نظام آمن لإدارة هويات المستخدمين وتسجيل الدخول.
- **خدمات الإشعارات (Notification Service):** نظام لإبلاغ المستخدمين ببدء المباريات ونتائج التوقعات (باستخدام WebSockets أو SSE).

## 3. أدوار المستخدمين والصلاحيات (User Roles & Permissions)
- **مستخدم مسجل (Registered User):**
  - تصفح جدول المباريات.
  - إدخال وتعديل التوقعات (قبل انطلاق المباراة المجدولة).
  - إنشاء والانضمام إلى دوريات خاصة (Private Leagues).
  - استعراض لوحة المتصدرين والترتيب الشخصي.
- **مدير نظام (Admin):**
  - إدارة بيانات الفرق، المجموعات، والمباريات.
  - تحديث وتأكيد نتائج المباريات النهائية (إذا لم يكن هناك ربط آلي API).
  - إدارة المستخدمين والدوريات لضمان نزاهة المنصة.
- **مستخدم زائر (Guest):**
  - تصفح جدول المباريات والترتيب العام فقط بصلاحيات للقراءة.

## 4. تصميم قاعدة البيانات (Database Design)
الكينونات الأساسية للنظام (Core Entities):
- **المستخدمين (Users):** `id`, `username`, `email`, `password_hash`, `total_points`, `created_at`.
- **المباريات (Matches):** `id`, `team_a_code`, `team_b_code`, `start_time`, `status` (لم تبدأ، جارية، انتهت), `actual_score_a`, `actual_score_b`, `stage`.
- **الفرق (Teams):** `id`, `name`, `group`, `flag_url`.
- **التوقعات (Predictions):** `id`, `user_id`, `match_id`, `predicted_score_a`, `predicted_score_b`, `points_earned`, `submitted_at`.
- **الدوريات (Leagues):** `id`, `name`, `creator_id`, `invite_code`.
- **أعضاء الدوريات (League_Members):** `league_id`, `user_id`, `joined_at`.

---

## 5. تدفق التطبيق (Application Flow)
1. **التسجيل / الدخول:** يقوم المستخدم بإنشاء حساب للوصول لخدمات التطبيق.
2. **استعراض المباريات:** تعرض الصفحة الرئيسية المباريات القادمة مصنفة.
3. **تقديم التوقع:** يختار المستخدم مباراة ويدخل النتيجة المتوقعة لكلا الفريقين ويحفظها (قبل التوقيت الرسمي للبداية).
4. **التنافس الاجتماعي:** يمكن للمستخدم تكوين دوري خاص (Mini-League) ومشاركة رمز الدعوة مع عائلته وأصدقائه.
5. **تحديث النتائج المباشرة:** مع انتهاء المباراة، يقوم مدير النظام أو نظام آلي بتحديث النتيجة لتصبح معتمدة.
6. **حساب وتوزيع النقاط:** يقوم سيرفر الـ Backend بإجراء مقارنة بين التوقعات والمركز الفعلي، وتوزيع النقاط (مثال: نقاط كاملة للنتيجة الدقيقة، نقاط أجزاء لتخمين الفائز).
7. **لوحة المتصدرين:** يتم تحديث ترتيب المستخدمين آلياً في القائمة العامة والدوريات الخاصة.

---

## 6. خريطة تجربة واجهة المستخدم (UI/UX Sitemap)
- **Home / Dashboard:** ملخص رصيد النقاط، ترتيب المستخدم الحالي، أبرز المباريات القادمة.
- **Fixtures (جدول المباريات):** استعراض جميع المباريات والنتائج والمواعيد.
- **My Predictions (توقعاتي):** شاشة مخصصة لإدارة التوقعات الحالية والسابقة.
- **Leagues (الدوريات المحتضنة):**
  - إنشاء دوري / الانضمام لدوري عبر كود الدعوة.
  - صفحة الدوري الداخلي (ترتيب الأعضاء المقارن).
- **Global Leaderboard (المتصدرين):** الترتيب العام لجميع اللاعبين المسجلين بالموقع.
- **Profile / Settings:** تعديل الحساب، إعدادات الإشعارات، وإحصائيات الدقة الشاملة.

---

## 7. الحزمة التقنية (Technical Stack)
- **الواجهة الأمامية (Frontend):** React.js (أو Next.js) مستخدمين TypeScript.
- **التنسيق والستايل (Styling):** Tailwind CSS لتجربة بصرية حديثة ومتجاوبة + Framer Motion للحركات (Animations).
- **الواجهة الخلفية (Backend):** Node.js إطارات (مثل Express او NestJS) بالـ TypeScript.
- **قاعدة البيانات (Database):** PostgreSQL (نظراً لكفاءتها العالية في العلاقات بين التوقعات والدوريات).
- **الاستضافة (Deployment):** Docker containers محمولة للواجهة وتعمل عبر Google Cloud Run أو ما يعادلها.
- **التوثيق (Authentication):** Firebase Auth للدخول الآمن والسريع وتعدد طرق الدخول.
